## What I Predict About the Theory's Shape

The AAD framework has a genuinely unusual structure: it's not trying to be a single unified derivation, but an *epistemic architecture* — a framework where the positive (separability), negative (identifiability floor), and constructive (additive coordinate forcing) halves name what's tractable, what's not, and where the coordinate system is forced. That three-part meta-structure is more sophisticated than most theoretical frameworks at this stage.

Section I is the most locked down — the persistence condition $\alpha > \rho/R$ is the load-bearing result, and everything else in that section is building up to it. I expect the naming targets for Section I to mostly have clean English names that work well (mismatch, update gain, tempo, adaptive reserve, chronica) because these concepts had years of TFT development before becoming AAD.

Section II (purposeful agents) is where I expect the most interesting naming questions. The strategy DAG formalism, the orient cascade, the satisfaction gap / control regret split — these are the AAD-specific contributions, and the names need to carry those distinctions clearly. Satisfaction gap / control regret is a named pair I already think works extremely well; my prediction is I'll vote strong keep on both.

Section III (composition) is still open enough that many names will be provisional or underspecified. I expect some targets here to be genuinely hard to vote on without reading the defining segments.

## Predictions About Specific Naming Questions

**Greek vocabulary (prolepsis, aisthesis, aporia, epistrophe, praxis):** I strongly expect to vote strong keeps on these. The README already makes the case for why they're load-bearing and what English alternatives they're competing against. The Greek cycle-phase vocabulary refuses the flatness of "predict/observe/mismatch/update/act" — that's exactly what good names should do.

**AAD and ASF acronyms:** I expect these are settled and correct. AAD was renamed from ACT precisely to avoid a collision in AI welfare literature, so the reasoning is documented and sound. ASF is the parent-level name for the framework. Both should be canonicalize-keep.

**Chronica:** I expect this to need a strong keep vote. It's the Greek-rooted term for the complete interaction history — the kind of concept that needs a distinctive noun to survive working-memory pressure. "Complete interaction history" is a description, not a name.

**Mismatch signal vs aporia signal:** Aporia is already one of the five cycle phase names, so naming the mismatch *signal* "aporia signal" would create a naming collision between the phase and the signal. I predict I'll vote against "aporia signal" for exactly this reason.

**Satisfaction gap / control regret:** Strong keeps. The 2×2 table works because both names are precisely evocative. These are the best-named pair in the project.

**Directed separation:** Strong keep. It's precise, memorable, and the "separation" metaphor correctly evokes the topological relationship (d-separation / Markov blanket) that makes it load-bearing.

**Orient cascade:** I need to understand this better from the defining segment before voting. But I predict it'll be a keep — it describes the sequential resolution order M_t → Σ_t → O_t accurately and memorably.

**Sector condition:** Interesting question. The name references a prior mathematical concept (sector condition in control theory) with good baggage — it's telling you the correction function lives in a sector, which is exactly right. But it might be confusing to non-control-theory readers. I'll wait for the defining segment.

**Class 1 / Class 2 / Class 3 agents:** I predict these will need rename votes. The numbering scheme is arbitrary. "Modular / integrated / partially coupled" or some variation is more informative and doesn't require a decoder ring.

**$U_M$ dual-use collision:** This is a real problem flagged in the tracker (same symbol for "model uncertainty" in single-agent and "epistemic unity" in multi-agent context). I predict I'll vote for disambiguation here — either $U_{\mathcal{M}}$ or $\Upsilon_M$ for the unity meaning, or careful canonicalization of context-appropriate names.

**Logogenic / logozoetic:** Strong keeps. These name structural properties precisely (constituted-by-language vs living-by-language with moral weight). The Greek register is appropriate.

**Adaptive reserve:** I'll wait for the defining segment, but I predict this is a strong keep. It names the margin between current operational state and the persistence threshold exactly.

**Hierarchy as a repeated word:** The tracker flags this — "hierarchy" appears in Pearl's causal hierarchy, convention hierarchy, correlation hierarchy, approximation tiering. This is too many uses of the same word, and I predict I'll want to vote for differentiation on some of these.

## What I Expect to Be Hard

The trickiest naming questions will be:
1. Symbol aliases (α₁, α₂, β, etc.) — these require understanding the sub-scope partition deeply enough to evaluate whether the English aliases are precise.
2. The meta-segment names (disc-separability-pattern, disc-identifiability-floor, disc-additive-coordinate-forcing) — these are cross-cutting meta-patterns and the names need to cover all their instances.
3. The [Concept] cluster targets where the underlying concept is described by a functional placeholder in the tracker — I'll need to read enough to know what the concept actually is.
